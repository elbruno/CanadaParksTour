## 6. Quick Start: How to Apply These Suggestions

### Activate GitHub MCP (High #1)

Edit `.copilot/mcp-config.json` and rename the key:

```jsonc
// Before:
"EXAMPLE-github": { ... }

// After:
"github": { ... }
```

### Create copilot-instructions.md (High #2)

Create `.github/copilot-instructions.md` with project standards:

```markdown
# Project: OntarioParksExplorer

## Architecture
- .NET Aspire orchestration with 3 services: api, blazor, react
- Service discovery uses lowercase names: `http://api`, `http://blazor`
- Three-layer architecture: Controllers → Services → EF Core DbContext
- AI features use IAiService abstraction (Copilot SDK, NOT OpenAI)

## Conventions
- DTOs for all API responses (never expose EF entities directly)
- AsNoTracking() on all read-only queries
- CORS policy "AllowFrontends" (dev: permissive, prod: lock down)
- Seed data from seed-data/parks.json (idempotent)
- xUnit for testing, Playwright for E2E
```

### Register Marketplace & Install Skills (High #4-6)

```bash
# Register awesome-copilot marketplace
squad plugin marketplace add github/awesome-copilot

# Install high-priority skills
squad plugin install copilot-sdk@awesome-copilot
squad plugin install ef-core@awesome-copilot
squad plugin install csharp-xunit@awesome-copilot

# Install medium-priority skills
squad plugin install dotnet-best-practices@awesome-copilot
squad plugin install containerize-aspnetcore@awesome-copilot
```

### Customize Project Conventions Skill (High #3)

Edit `.squad/skills/project-conventions/SKILL.md` — replace placeholder content with the actual conventions from this project (see copilot-instructions.md content above as a starting point).

### Add Playwright MCP (Medium #10)

Add to `.vscode/mcp.json`:

```json
{
  "servers": {
    "playwright": {
      "command": "npx",
      "args": ["@anthropic/playwright-mcp-server"]
    }
  }
}
```
